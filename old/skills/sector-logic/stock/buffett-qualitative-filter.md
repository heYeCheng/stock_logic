# Buffett 定性过滤器

**Version**: 1.0  
**Created**: 2026-04-17

---

## 核心定义

Buffett 定性过滤器是对个股进行定性分析的强制检查环节，用于识别无法通过量化指标捕捉的重大风险。

**原则**：定性因素存在重大缺陷的股票，无论量化分数多高，都应降级或排除。

---

## 检查维度

### 1. 护城河检查

**检查项**：
- [ ] 是否具有品牌优势（消费者愿意为品牌支付溢价）
- [ ] 是否具有成本优势（规模效应、独特工艺、地理位置）
- [ ] 是否具有网络效应（用户越多价值越大）
- [ ] 是否具有转换成本（客户更换供应商成本高）
- [ ] 是否具有特许经营权或专利壁垒

**判定**：
- **pass**: 至少 2 项为"是"
- **pass_with_concerns**: 仅 1 项为"是"
- **fail**: 全部为"否"

---

### 2. 管理层检查

**检查项**：
- [ ] 管理层是否稳定（近 3 年核心高管离职率 < 30%）
- [ ] 管理层是否诚信（无财务造假、违规记录）
- [ ] 管理层是否专注（主业清晰，不盲目多元化）
- [ ] 股权激励是否与股东利益一致
- [ ] 资本配置能力是否优秀（历史并购、分红、回购记录）

**判定**：
- **pass**: 至少 4 项为"是"
- **pass_with_concerns**: 2-3 项为"是"
- **fail**: 0-1 项为"是"或存在诚信问题

---

### 3. 财务健康检查

**检查项**：
- [ ] 资产负债率是否 < 60%（金融股除外）
- [ ] 经营现金流是否持续为正（近 3 年至少 2 年为正）
- [ ] 是否存在大额商誉减值风险（商誉/净资产 < 30%）
- [ ] 是否存在大额应收账款风险（应收账款/营收 < 50%）
- [ ] 是否存在大额存货减值风险（存货/营收 < 50%）

**判定**：
- **pass**: 至少 4 项为"是"
- **pass_with_concerns**: 2-3 项为"是"
- **fail**: 0-1 项为"是"或存在重大减值风险

---

### 4. 业务可理解性检查

**检查项**：
- [ ] 业务模式是否简单易懂
- [ ] 收入来源是否清晰（前五大客户占比 < 50%）
- [ ] 是否在能力圈内（分析师能够理解其商业模式）
- [ ] 是否存在复杂关联交易

**判定**：
- **pass**: 至少 3 项为"是"
- **pass_with_concerns**: 2 项为"是"
- **fail**: 0-1 项为"是"或存在复杂关联交易

---

## 综合判定

| 维度 | 权重 | 说明 |
|------|------|------|
| 护城河 | 0.3 | 长期竞争优势 |
| 管理层 | 0.3 | 执行力和诚信 |
| 财务健康 | 0.25 | 风险控制 |
| 业务可理解性 | 0.15 | 能力圈内 |

**最终判定**：
- **pass**: 所有维度至少 "pass_with_concerns"，且至少 2 个维度 "pass"
- **pass_with_concerns**: 所有维度至少 "pass_with_concerns"，但只有 0-1 个维度 "pass"
- **fail**: 任一维度 "fail"

---

## 强制降级规则

1. **buffett_filter = fail** → 强制归入"其余"层，无论 stock_thesis_score 多高
2. **buffett_filter = pass_with_concerns** → 最高只能归入"观察名单"层
3. **buffett_filter = pass** → 可正常参与分层

---

## 使用说明

本过滤器由 LLM 或规则引擎执行，输入为个股的定性信息，输出为 pass/pass_with_concerns/fail。

**LLM Prompt 示例**：
```
请根据以下信息，对 {stock_name} 进行 Buffett 定性过滤器检查：

- 公司信息：{company_description}
- 财务数据：{financial_metrics}
- 管理层信息：{management_info}
- 业务信息：{business_info}

请按以下维度逐一检查：
1. 护城河（品牌/成本/网络效应/转换成本/特许经营权）
2. 管理层（稳定性/诚信/专注度/股权激励/资本配置）
3. 财务健康（负债率/现金流/商誉/应收账款/存货）
4. 业务可理解性（业务模式/收入来源/能力圈/关联交易）

输出格式：
{
  "moat": "pass/pass_with_concerns/fail",
  "management": "pass/pass_with_concerns/fail",
  "financial_health": "pass/pass_with_concerns/fail",
  "understandability": "pass/pass_with_concerns/fail",
  "final_result": "pass/pass_with_concerns/fail",
  "reasoning": "..."
}
```
