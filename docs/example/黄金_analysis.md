## 系统观察（黄金板块特有）

**1. 黄金板块适合做系统的"对抗性测试集"。** 宁德那轮是一致看多，所有指标共振向上；黄金是典型的多空激烈对抗 + 板块冷却期，正好可以验证系统在"看上去有机会但市场不配合"场景下是否会盲目给出重仓信号。**结果是：系统正确地给出了半仓而非重仓**，市场层起了关键作用。

**2. "扩散_逆势"这个结构标签非常关键。** 板块整体在跌，但业绩股在涨——这是黄金当前的真实形态。Phase 0.5 能捕捉到这个状态是加分项；但执行层目前没有针对"扩散_逆势"的特殊处理（只有"快速轮动"会触发 0.80 收紧）。建议 Phase 2 加入"逆势"状态下的风控规则（比如只配置业绩明确兑现的个股）。

**3. 黄金板块的 macro_multiplier 应用是错位的。** 系统当前的 macro_multiplier 是基于**国内宏观**（PMI、M2）的，但黄金最敏感的是**全球宏观**（美联储、美元指数）。按逻辑族映射表，黄金主导逻辑是"流动性驱动"+"事件驱动"，全球联动敏感度极高（+0.9），国内流动性敏感度弱。Phase 0.5 把这个差异委托给 L1 的逻辑强度来消化（美联储降息延后直接以 negative logic 形式出现，strength = 0.80），理论上是对的，但实操上存在"宏观乘数对黄金板块是个冗余变量"的风险——黄金几乎等同于 macro_multiplier = 1.0 永远不变。这不是 bug 但值得在 Phase 2 考虑给不同逻辑族应用不同 multiplier。

**4. 波动率 99 分位是系统没捕捉到的红灯。** 1 月底 30+ 只黄金股单日跌停、CBOE 金 ETF 隐含波动率处于 2009 年来 99% 分位——这是一个极其明确的尾部风险信号。目前系统只能通过 L2 情绪分的 z-score 间接捕捉，但 z-score 会随时间滚动被"新常态"稀释。**建议引入独立的"波动率极值检测模块"**——当某板块的隐含波动率/ATR 处于历史 95 分位以上时，直接将仓位再乘 0.8 折扣。这不需要 LLM，纯规则引擎即可。

**5. 两只个股打分几乎一样但风险属性不同。** 紫金（0.610）和赤峰（0.614）仓位分数打平，但赤峰的潜在回撤幅度会明显大于紫金（1月教训：小盘黄金股集体跌停时大蓝筹跌幅温和得多）。**系统没有给出这个差异**——因为 ATR 虽然扩张但还没到极值，stock_market_score 也没有显著分化。这是 logic_match_score 和 affiliation_strength 没有充分反映"波动率暴露"的结果。Phase 2 应该在 stock_composite_score 中引入 **下行风险调整**（比如 beta、最大回撤）。

**跑盘结论**：黄金板块在 Phase 0.5 系统下给出 **"半仓"** 信号（代表个股均 position_score ≈ 0.61）。核心驱动是 **央行购金 + 地缘避险 + 业绩兑现**，核心压制是 **美联储降息延后 + 板块情绪冷却 + 高位波动率**。系统同时标记 `has_headwind = True` 和 `market_state 接近 weak`，暗示这是一个**右侧未确认、需要等待的机会**，而不是像宁德那样可以加仓追进。具体执行上：**紫金作为压舱石优先配置（波动更小）、赤峰作为弹性头寸控制仓位（高波动双刃剑）、等美伊谈判和4月29日FOMC落地再决定加仓还是减仓**。
