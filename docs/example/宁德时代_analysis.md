## 系统观察（跑盘过程中暴露的问题）

作为系统冷静审视这一次跑盘，有几点值得记录：

**1. 龙头判定规则对超大市值失效。** 宁德单日成交285亿、占板块30%+吸金，但"近5日需有涨停"这个规则卡住它——对万亿市值的权重龙头，大涨5%已是巨量资金推动，不可能频繁涨停。建议 Phase 2 为超大市值股加入例外规则（如"市值板块前3且成交前3"直接判定龙头）。

**2. logic_match_score 对龙头的对冲能力识别依赖 LLM 质量。** 宁德对碳酸锂涨价的对冲（价格联动+长协+钠电+上游300亿资源集团）是其相比二线电池厂的核心 alpha，LLM 必须准确识别这一点并给出 0.3 而非 0.8 的 match_score，否则反向逻辑会被过度放大。这是系统对 LLM 理解能力的高要求点。

**3. 净推力截断到 1.0 丢失了信息。** 宁德六条逻辑正向累加本来 2.90，截断后和一个"只有两条正向"的板块龙头打平，辨识度下降。Phase 0.5 的做法是正确的（防溢出），但排序时应保留未截断的 raw 值做二次排序。

**4. 市场状态 0.75 已站在悬崖边。** 如果 4.21 科技日消息兑现后出现跳空高开放巨量，情绪 z-score 很可能一步跨过 1.5，这时系统会在下一个交易日自动从"重仓"信号切换到"谨慎/减仓"。**这是系统设计得好的地方**——事件驱动的狂热不会一直支撑"重仓"判断。

**5. has_headwind 的 0.85 折扣给得恰到好处。** 不加这个折扣，position_score = 0.898 对应"重仓"；加了折扣 = 0.763 仍"重仓"但逼近半仓边界，既保留正向判定、又体现了反向逻辑的真实风险。
