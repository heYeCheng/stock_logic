# 交易逻辑驱动的智能选股系统——设计思想与架构详解（评审版）
> **版本**: v7.1 评审版  
> **日期**: 2026-04-19  

---

## 1. 系统要解决的核心问题

A股波段交易中，个人投资者常陷入三种困境：

1. **看对方向但选错标的**：知道某个板块有机会，但买入的个股并未受益于核心逻辑，甚至逆势下跌。
2. **被正常波动震出局**：无法区分“逻辑反转”与“技术性回调”，在洗盘中丢失筹码。
3. **信息过载与叙事噪音**：面对海量新闻和研报，无法提炼出当前驱动行情的核心矛盾，追涨杀跌。

本系统不是另一个技术指标扫描器，也不是对新闻进行简单情感分析的工具。它是一个**逻辑驱动的决策辅助系统**，试图通过结构化的框架回答三个递进问题：

- **逻辑层**：当前市场到底在交易什么故事？这个故事本身的真伪和强度如何？
- **市场层**：市场有没有用真金白银确认这个故事？交易进行到了什么阶段？
- **执行层**：在当前位置，这个故事还有多少未被定价的空间？应该用多大仓位参与？如果错了，如何体面退出？

## 2. 总体架构：五层模型与数据隔离铁律

系统按照“宏观环境 → 板块逻辑 → 板块市场 → 个股承接 → 执行决策”五层组织，数据流向严格单向，确保每一层的职责清晰且互不污染。

```
┌─────────────────────────────────────────────────────────────┐
│  L0 宏观环境层                                               │
│  基于月度宏观指标和政策事件，输出一个全局乘数（0.85~1.15），  │
│  对所有板块的仓位建议进行统一的加减分调整。                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  L1 板块逻辑层                                               │
│  输入：产业政策、公司财报、供需数据、全球宏观因子。           │
│  （严格禁止输入任何价格、涨跌幅、成交量信息）                 │
│  输出：每个板块当前所有活跃的交易逻辑（正向/反向），每条逻辑  │
│        的强度、逻辑族、净推力、以及是否存在强反向压制。       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  L2 板块市场层                                               │
│  输入：价格、成交量、涨停家数、换手率、板块广度等纯量价数据。 │
│  （严格禁止输入任何政策、财报、研报等基本面信息）             │
│  输出：板块的综合市场强度、市场状态（弱势/正常/过热）、以及   │
│        内部结构（领涨是聚焦于少数龙头还是全面扩散）。         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  L3 个股承接层                                               │
│  输入：个股的多板块归属关系、主营业务描述、以及个股自身的     │
│        量价行为。                                             │
│  输出：个股对板块各项逻辑的受益/受损程度（逻辑受益度）、个股  │
│        自身的市场强度、以及综合两者后的最终评分。             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  L4 执行决策层                                               │
│  融合以上所有层的输出，通过连续仓位函数给出操作建议，并进行   │
│  A股交易约束检查（涨跌停、T+1、停牌）。                       │
│  输出：方向（看多/观望）、建议仓位（重仓/半仓/轻仓/观察）、   │
│        止损参考条件、等待条件。                               │
└─────────────────────────────────────────────────────────────┘
```

**数据隔离铁律**是系统稳定性的基石：
- L1（逻辑层）的输入中**不包含任何价格、涨跌幅、成交量信息**。逻辑强度的变化只能由非价格证据（政策、财报、产业数据）触发，杜绝“因为涨了所以逻辑强”的循环论证。
- L2（市场层）的输入中**不包含任何政策文件、研报内容**。市场热度只看量价行为，杜绝“因为故事好听所以追高”的情绪干扰。
- L4（执行层）可以消费以上所有输出，但**绝不反向修改**任何一层的评分。

## 3. 各层核心设计详解

### 3.1 宏观环境层（L0）：全局加减分器，不做总闸门

宏观层的作用是提供背景温度，而非直接决定能否交易。

- **四象限框架**：采用“货币-信用”框架，根据PMI（增长动能）和M1-M2剪刀差/Shibor（流动性动能）将宏观环境分为四象限：宽信用期、紧流动性期、双紧期、宽货币期。每个象限对应一个基准乘数（例如宽信用期=1.10，双紧期=0.90）。
- **事件触发微调**：在月度数据真空期，若发生央行降准、政治局会议定调转向等重大事件，系统允许手动录入临时调整值（如+0.05），叠加到基准乘数上。
- **最终输出**：一个范围在 **0.85~1.15** 之间的全局乘数 `macro_multiplier`，在执行层对所有板块的仓位分进行统一缩放。这意味着宏观环境极度友好时，仓位可适度放大；宏观环境严峻时，仓位整体收缩，但不会因此完全否决任何结构性机会。

**未来演进方向**：不同逻辑族（如产业趋势 vs. 流动性驱动）对宏观维度的敏感度天然不同。例如，产业趋势对流动性不敏感，而券商板块（流动性驱动）极其敏感。后续可引入**逻辑族级别的宏观敏感度映射**，用约20个参数替代上百个板块参数，实现更精细的宏观调整。

---

### 3.2 板块逻辑层（L1）：多逻辑并发 + 事件计分板

这是系统的“大脑”，负责回答“市场在交易什么故事”。

#### 3.2.1 为什么需要多逻辑并发

一个板块在任何时刻都很少只受单一逻辑驱动。以黄金板块为例，其价格同时受到以下逻辑影响：

| 逻辑 | 方向 | 逻辑族 | 作用 |
|------|------|--------|------|
| 全球央行降息预期升温 | 正向 | 流动性驱动 | 主推力 |
| 地缘冲突引发避险需求 | 正向 | 事件驱动 | 辅助推力 |
| 美元指数走强 | 反向 | 流动性驱动 | 压制力 |

传统的“一个板块一个叙事”模型无法描述这种复杂的多空博弈。本系统显式管理每一条活跃逻辑，并计算**净推力**（正向合力 - 反向合力）和**分歧程度**。当存在强反向逻辑时，系统不会简单看空，而是收紧风控（如仓位降档、止损收窄），以应对更大的波动性。

#### 3.2.2 如何客观评估逻辑强度：事件计分板（核心创新）

传统做法中，逻辑强度评估存在两大痛点：
- **LLM直接打分**：输出极不稳定，同一输入多次运行分数可差0.3以上，无法用于量化决策。
- **新闻热度统计**：新闻爬虫覆盖度不可控，且新闻热度与股价高点同步，是典型的滞后指标。

本系统采用 **“LLM提取结构化事件 + 规则引擎计分”** 的分离式架构：

**第一步：LLM事件提取**
- 每日收盘后，系统将当日与板块相关的新闻、公告、数据摘要拼接为文本，输入LLM。
- LLM的任务**不是**打分，而是提取**结构化事件**。例如：
  ```json
  {
    "event_type": "国家级政策发布",
    "direction": "positive",
    "summary": "工信部等七部门联合印发《推动算力基础设施高质量发展行动计划》",
    "event_date": "2026-04-19"
  }
  ```

**第二步：规则引擎计分**
- 系统预定义了每种事件类型的**加分值和有效期**。例如：
  - 国家级政策发布：+0.20（有效期20个交易日）
  - 龙头公司业绩超预期：+0.15（有效期10个交易日）
  - 政策收紧/监管问询：-0.25（有效期15个交易日）
- 规则引擎每日扫描事件表，计算每条逻辑的当前强度 = 初始强度 + Σ(有效事件加减分) - 自然衰减。
- 强度被限制在 **[0.15, 0.95]** 之间。

**第三步：自然衰减**
- 若一条逻辑长时间无新事件，其强度会逐渐衰减。衰减速度与逻辑族挂钩：
  - **快逻辑**（事件驱动、政策驱动）：每5个交易日衰减0.05（每日约0.01）。事件降温后应快速遗忘。
  - **慢逻辑**（产业趋势、流动性驱动）：每10个交易日衰减0.02（每日约0.002）。长线逻辑不应因短期无新闻而快速失效。

**这套机制的价值**：
- **可复现**：相同事件输入必然得到相同强度输出，解决了LLM的随机性问题。
- **可回测**：只需回放历史事件记录，即可验证强度曲线是否与股价有预测关系。
- **可解释**：每一条强度变化都能追溯到具体事件，便于复盘和迭代规则。

#### 3.2.3 AI如何参与模糊判断：定性判断 + 经验锚点模式

对于“一条新逻辑有多重要”、“个股在产业链中受益程度”这类难以纯规则化的判断，系统采用 **“LLM定性 + 结构化锚点”** 的协作模式。

以**逻辑重要性判断**为例：
1. 系统在Prompt中提供一份**经验锚点**（存放于独立配置文件，可版本管理）：
   - **高重要性**：该逻辑能解释板块内多数龙头股近期走势，有≥3条独立证据，市场共识度高。
   - **中重要性**：能解释部分龙头股走势，有1-2条独立证据。
   - **低重要性**：仅影响个别个股，证据单薄。
2. LLM阅读上下文信息后，判断当前逻辑落在哪个锚点区间，输出**定性等级**（高/中/低），**不输出具体分数**。
3. 规则引擎将定性等级映射为**初始强度参考值**（如高→0.7，中→0.5，低→0.3），后续由事件计分板接管动态调整。

这种模式保留了LLM在非结构化信息理解上的灵活性，同时用结构化锚点锁死了输出的方差，避免了LLM直接打分的灾难。所有锚点均可独立迭代，不影响核心代码。

### 3.3 板块市场层（L2）：纯量价认知

市场层的任务是回答“市场有没有在用资金确认逻辑”，完全基于量价数据。

#### 3.3.1 市场雷达：技术与情绪二维评分

- **技术面（0-10分）**：综合评估**趋势结构**（均线排列与乖离）、**量能配合**（近5日量比）、**板块广度**（收盘价站上20日线的个股比例）和**相对全市场强度**。一个健康的上涨板块通常呈现“价涨量增、多数个股参与、跑赢大盘”的特征。
- **情绪面（0-10分）**：**刻意避免使用新闻爬取热度**，完全采用市场行为代理：**涨停热度**（近5日涨停家数占比）、**连板高度**（板块内最高连板天数）、**换手活跃度**（平均换手率近5日/近20日比值）。这些数据每日收盘后稳定可得，且真实反映了资金的追逐意愿。

两者加权合成为 **0~1** 的板块市场强度分。

#### 3.3.2 三状态判定

将连续的市场强度分映射为三个离散状态，用于执行决策：

- **弱势（weak）**：强度分 < 0.4。市场关注度不足或处于下跌趋势，**不建议参与**。
- **正常（normal）**：强度分在 0.4~0.7 之间。市场处于正常交易状态，可根据逻辑层信号正常参与。
- **过热（overheated）**：强度分 > 0.7，且情绪分显著高于历史均值（用z-score > 1.5 判断，消除不同板块的情绪基线差异）。此时市场情绪高涨，风险积聚，**应警惕减仓**。

#### 3.3.3 领涨集中度：自动识别板块内部结构

传统做法需要人工将板块划分为若干子分支（如通信→光模块/运营商/设备商），维护成本极高。本系统用**领涨集中度**这个纯量价指标来代理：

- **计算方式**：取板块内近5日涨幅前20%的个股，计算它们的成交额之和占板块总成交额的比例。
- **解读**：
  - **集中度 > 60%**：资金高度聚焦于少数领涨股，板块内部呈现 **“聚焦”** 结构。此时应优先配置领涨分支的龙头股。
  - **集中度 < 40%**：资金分散，板块普涨，呈现 **“扩散”** 结构。此时可配置走势稳健的中军股。
  - **领涨股重叠率 < 30%**：每日领涨的股票都在换，呈现 **“快速轮动”** 结构。说明缺乏持续性，应谨慎或仅轻仓试探。

这个指标无需任何人工维护，自动适应板块内部结构的变化。

### 3.4 个股承接层（L3）：多板块归属与暴露计算

个股层的任务是精确刻画每只股票对当前市场逻辑的受益程度。

#### 3.4.1 解决“一票多板块”问题

A股中，一只股票往往同时属于1个申万行业板块和若干个概念板块。例如“新易盛”既属于通信设备，又属于CPO、AI算力、5G。系统维护**个股-板块映射表**，并为每个映射赋予**归属强度**：
- **1.0**：行业板块、或市场公认的主营概念（如CPO）。
- **0.8**：强关联概念，个股有相关产品线但非主营。
- **0.5**：弱关联概念，仅因业务沾边或历史原因被纳入。

#### 3.4.2 计算暴露系数

对于每只个股，系统遍历其所有归属板块的每一条活跃逻辑，计算**暴露系数**：

```
暴露系数 = 归属强度 × 业务匹配度
```

**业务匹配度**由两部分构成：
- **关键词匹配**：为每个板块维护一个自动生成的核心关键词库（例如CPO：光模块、光引擎、硅光）。通过匹配个股主营业务描述中的关键词，给出高/中/低匹配分。
- **产业链环节定位**：由LLM一次性判断个股在产业链中的受益程度（直接受益/间接受益/边缘受益/不受益），并缓存结果。财报季或重大重组后自动刷新。

个股最终的**逻辑受益度**（0-100分）是其对所有正向逻辑暴露系数之和减去对所有反向逻辑暴露系数之和，经归一化后得出。这个分数直接回答了：“在当前市场交易的所有逻辑中，这只股票到底能沾多少光？”

#### 3.4.3 个股地位自动识别

系统完全自动化识别个股在板块内的角色：

- **龙头**：近10日涨幅居板块前10%、近5日内有过涨停、且业务匹配度≥0.7。特征是领涨、高弹性。
- **中军**：总市值和近20日日均成交额均位于板块前5名。特征是大资金战场、走势稳健。
- **跟风**：其余个股。

**蓄力中军机制**：当板块市场状态为“正常”或“过热”，但某中军个股的自身市场强度分却 < 0.5（即还没怎么涨）时，系统会为其打上“⚡蓄力”标签，提示用户关注潜在的补涨机会。Phase 0.5阶段，这仅作为一个提示标签，不直接修改评分，避免人工干预评分逻辑。

#### 3.4.4 催化剂标记

系统放弃获取成本高昂的“业绩超预期一致预期”、“合同金额NLP提取”等数据，仅使用收盘后稳定可得的公开信息作为持有信心标记：
- **强**：近3日有涨停，或近5日涨幅>20%且量比>1.5。
- **中**：近5日登上龙虎榜且机构席位净买入。
- **无**：以上皆无。

催化剂标记**不参与量化排序**，仅在展示时提供参考，增强用户对趋势股（沿均线稳步上涨但无涨停）的持仓信心。

### 3.5 执行决策层（L4）：连续仓位函数与约束检查

#### 3.5.1 连续仓位函数

传统做法常用离散的决策矩阵（例如：净推力>0.3且市场=确认→重仓）。这种方法存在**硬阈值断裂**问题：净推力0.29和0.31的微小差异，会导致“观察”和“重仓”的截然不同，参数极度过拟合。

本系统采用**连续仓位函数**：
1. 将**净推力**（归一化到0-1）、**个股综合评分**、**板块市场强度分**按预设权重（如0.4/0.4/0.2）加权求和。
2. 乘以宏观乘数 `macro_multiplier`，得到连续的 `position_score`（0~1之间）。
3. 若存在强反向逻辑，`position_score` 乘以0.85（仓位降一档）；若板块为“快速轮动”结构，再乘以0.80。
4. 将连续的 `position_score` 映射到离散的仓位建议：
   - **> 0.75** → 重仓
   - **0.50 ~ 0.75** → 半仓
   - **0.30 ~ 0.50** → 轻仓
   - **< 0.30** → 观察

#### 3.5.2 止损与持有决策

持有决策独立于进入决策，每日运行：

- **逻辑弱化**：若主逻辑的强度从持仓期间的最高点回撤超过0.15，触发减仓。这比与“建仓日”比较更合理——即使当前强度仍高于建仓日，但较高点已显著回落，同样意味着驱动力衰减。
- **市场破位**：板块市场状态降级为“弱势”，或个股收盘价跌破关键均线（如20日线）且连续两日未能收回，触发退出。
- **主线切换**：原板块的领涨地位被新板块取代，且原逻辑热度同步下降，触发重新评估。

#### 3.5.3 A股交易约束检查

在输出推荐前，系统自动过滤不可执行的标的：
- 当日主板/中小板涨停（+10%）：不推荐买入，提示“等待分歧”。
- 当日科创板/创业板涨停（+20%）：不推荐买入，提示“极高风险，等待分歧”。
- 近5日累计涨幅>30%且非涨停：仓位建议自动降一档，提示“追高风险”。
- 停牌、当日跌停：直接跳过或标记无法交易。

## 4. 系统如何回答“为什么持有，为什么卖出”

- **为什么继续持有**：
  1. 主逻辑的强度（来自事件计分板）未出现连续衰减，且无明确负面事件。
  2. 板块市场状态维持在“正常”或“过热但未破位”，技术面未发出卖出信号。
  3. 个股自身的综合评分稳定，未出现显著的逻辑匹配度恶化（如公司公告业务转型剥离了核心概念）。

- **为什么减仓/卖出**：
  1. **逻辑证伪**：事件计分板捕获到针对主逻辑的明确负面事件（如政策收紧、龙头业绩暴雷），导致逻辑强度大幅下降。
  2. **市场过热**：板块进入“过热”状态，且出现放量滞涨或领涨股集体退潮。
  3. **技术破位**：板块或个股跌破关键趋势支撑（如MA20、MA60），且伴随成交量放大，市场用真金白银投了反对票。
  4. **主线切换**：市场涌现出强度更高、逻辑更清晰的新主线，导致原板块资金持续流出。

卖出信号由**市场层触发（破位）或逻辑层触发（证伪）**，杜绝了“我觉得涨不动了”的主观臆断，从而帮助用户规避情绪化交易。

## 5. 完整降级链路：系统的韧性设计

任一数据源或服务都可能失效。系统设计了完整的降级矩阵，确保核心功能持续运行：

| 失效模块 | 降级行为 |
|----------|----------|
| 涨停板数据无法获取 | 情绪面评分仅使用换手活跃度单项，并收紧过热判定标准（z-score阈值从1.5提高至2.0）。 |
| 板块成分股列表获取失败 | 跳过该板块，当日不参与推荐。 |
| 新闻/事件数据源中断 | 事件计分板停摆，逻辑强度沿用上一日并按正常衰减速度自动衰减，系统临时降级为“市场层驱动”。 |
| 北向资金数据缺失 | 资金面评分置空，市场综合强度完全由技术和情绪二维计算。 |
| LLM服务不可用 | 逻辑层完全沿用上一日数据，强度按自然衰减规则执行，系统降级为“市场层驱动”。 |
| 宏观数据连续两个月缺失 | 宏观乘数固定为1.00，宏观层暂时失效。 |

每日运行前进行核心数据源（日线、板块成分）的健康检查，若失效则发送告警并中止，避免输出错误推荐。

## 6. 回测方案设计

事件计分板的有效性需要历史数据验证。由于无法逐日调用LLM回放历史事件，Phase 0.5 采用**手工标注 + 回放**的折中方案：

1. 选取3~5个典型板块（如黄金、半导体、医药）。
2. 由研究员根据已知的重大事件节点（如政策发布、财报季），手工标注过去6个月的逻辑强度变化曲线（分段赋值，中间线性插值）。
3. 回测时，用标注好的强度曲线替代实时事件计分板，验证市场层和仓位函数的有效性。
4. Phase 2 在积累足够历史新闻文本后，可批量调用LLM提取事件，实现全自动事件回测。

## 7. 供讨论的开放问题

本设计历经多轮迭代，但仍欢迎就以下方向提出专业意见：

1. **事件计分板的规则粒度**：当前约10种事件类型，是否足以覆盖A股的主要催化形式？是否需要为不同行业引入差异化权重？
2. **宏观层的演进路径**：从“全局乘数”演进到“逻辑族敏感度映射”的性价比如何？是否有更简洁有效的表达方式？
3. **LLM定性锚点的校准**：锚点由行业经验设定，如何通过回测或实盘反馈系统化地校准这些锚点（如“高重要性”的初始强度应该映射为0.7还是0.8）？
4. **极端行情适应性**：在系统性流动性危机中，市场层和逻辑层可能同时失效，是否需要设计一个独立的“危机模式”来强制空仓？
5. **仓位函数权重的最优设定**：逻辑、个股、市场三者的权重（0.4/0.4/0.2）如何通过滚动回测科学确定，而非依赖经验预设？
