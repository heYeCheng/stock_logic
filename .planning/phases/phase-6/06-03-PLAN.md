## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API | User submits override values; backend must validate ranges and reason length |
| API → Database | Override values written to DB; audit trail ensures accountability |
| User → Form | User may attempt invalid values; frontend and backend both validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Tampering | Override API | mitigate | Pydantic v2 Field constraints (ge=0, le=1); reason min_length=10; backend validates even if frontend bypassed |
| T-06-03-02 | Spoofing | Override API | mitigate | Depends(get_current_user) requires authentication; override_by field captures identity |
| T-06-03-03 | Repudiation | Audit trail | mitigate | OverrideAuditLog is append-only (no update/delete endpoints); immutable record of all changes |
| T-06-03-04 | Information Disclosure | Override history | accept | Only authenticated users can view; no sensitive data exposed (strength values are internal metrics) |
| T-06-03-05 | Denial of Service | Override flood | accept | Rate limiting handled at gateway level; low-value target for DoS |
| T-06-03-06 | Elevation of Privilege | Unauthorized override | mitigate | Role-based access control: only users with 'analyst' or 'admin' role can create overrides (future enhancement) |
</threat_model>
