Error handling:

- ConfigCommitError: Commit operation failed
- ConfigRevertError: Revert operation failed

<name>Task 3: Create YAML config API endpoints</name>
<files>src/api/v1/endpoints/config_yaml.py</files>
<action>
Create FastAPI router for YAML config operations.

Implement endpoints:
1. GET /api/v1/config/yaml/{config_file}: Get current config content
   - Returns: ConfigContentResponse
   - Reads file from repo, returns current HEAD content

2. PUT /api/v1/config/yaml/{config_file}: Update config with commit
   - Request: ConfigUpdateRequest
   - Returns: {commit_sha: str, message: str}
   - Validates YAML syntax before commit
   - Creates git commit with author info

3. GET /api/v1/config/yaml/{config_file}/history: Get commit history
   - Query: limit (default 20)
   - Returns: CommitHistoryResponse

4. GET /api/v1/config/yaml/{config_file}/diff: Get diff between versions
   - Query: old_sha, new_sha (or use HEAD~1)
   - Returns: ConfigDiffResponse with unified diff

5. POST /api/v1/config/yaml/{config_file}/revert: Revert to previous version
   - Request: ConfigRevertRequest
   - Returns: ConfigRevertResponse
   - Creates new commit reverting changes

6. POST /api/v1/config/yaml/validate: Validate YAML syntax
   - Request: ConfigValidationRequest
   - Returns: ConfigValidationResponse
   - Uses PyYAML to validate syntax

Follow patterns from system_config.py endpoint structure.
</action>
