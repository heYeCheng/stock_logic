<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React frontend → FastAPI API | Untrusted input (query params, stock codes) crosses here |
| FastAPI → MySQL Database | SQL injection risk if queries not parameterized |
| Environment → CORS config | Misconfigured CORS allows unauthorized frontend access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Injection | All endpoints | mitigate | SQLAlchemy 2.0 ORM with parameterized queries (no raw SQL) |
| T-06-01-02 | Tampering | Query params | mitigate | Pydantic v2 validation with `Field(ge=0, le=100)` constraints |
| T-06-01-03 | Spoofing | CORS config | mitigate | Explicit origin allowlist, `CORS_ALLOW_ALL=false` by default |
| T-06-01-04 | Information Disclosure | Error responses | mitigate | Generic error messages, detailed logs server-side only |
| T-06-01-05 | Tampering | Stock code path param | mitigate | Regex validation `^[0-9]{6}\.(SZ|SH)$` in endpoint |
</threat_model>
